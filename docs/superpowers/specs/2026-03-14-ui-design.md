# Jet Holdem - UI Implementation Design

## Overview

ゲームエンジン（Plan 1）完成済みの状態で、ローカルモード（BLEなし）で動作するUIを構築する。将来のBLE通信（Plan 2）追加時にUIを変更せずにGameServiceの実装を差し替えるだけで対応できる設計とする。

**スコープ:** Plan 3（UI）— Plan 2（BLE）・Plan 4（データ永続化）より先に実装する。

---

## 1. 技術スタック

| 項目 | 選定 | 理由 |
|---|---|---|
| ナビゲーション | Expo Router | Expoプロジェクト標準、ファイルベースルーティング |
| 状態管理 | React Context + useReducer | 外部ライブラリ不要、アプリ規模に十分 |
| テスト | React Native Testing Library + Jest | 既存Jest環境を活用 |
| スタイリング | React Native StyleSheet | 標準API、追加依存なし |

## 3. GameServiceインターフェース

UIとゲームロジックを疎結合にする抽象層。将来BleGameServiceに差し替え可能。

```typescript
// src/services/GameService.ts

interface GameService {
  // 状態取得
  getState(): GameState;

  // アクション
  startGame(playerNames: string[], blinds: Blinds, initialChips: number): void;
  startRound(): void;
  handleAction(seat: number, action: PlayerAction): ActionResult;
  resolveShowdown(): ShowdownResult;
  prepareNextRound(): void;

  // 状態変更の購読
  subscribe(listener: (state: GameState) => void): () => void;
}
```

**LocalGameService:**
- GameLoopをラップし、全メソッドをローカルで処理
- subscribe呼び出し時、アクション後に自動でlistenerに最新stateを通知

**将来のBleGameService:**
- BLE経由でGameState Notifyを受信 → subscriberに通知
- handleAction → BLE Write でホストに送信
- startGame/startRound等はホスト側のみ有効

## 4. GameContext

```typescript
// src/contexts/GameContext.tsx

interface GameContextValue {
  state: GameState | null;
  mode: 'hotseat' | 'debug';
  viewingSeat: number;           // 現在の視点プレイヤー
  service: GameService;
  dispatch: (action: GameAction) => void;
}
```

- `mode`: ロビーで選択、ゲーム中は変更不可
- `viewingSeat`: ホットシートモードではアクティブプレイヤーに追従、デバッグモードでは固定（seat 0）

## 6. GameScreen レイアウト

flexboxベースでレスポンシブ対応。座標は比率ベース。

```
┌─────────────────────────────────┐
│  ステータスバー                    │
├─────────────────────────────────┤
│                                   │
│         Seat 2 (対面)             │  flex: 1
│                                   │  テーブル楕円エリア
│  Seat 1 (左)     Seat 3 (右)     │
│                                   │
│      ┌─────────────────┐        │
│      │ ポット / コミュニティ │        │
│      └─────────────────┘        │
│                                   │
│         Seat 0 (自分)             │
├─────────────────────────────────┤
│  アクションボタン                  │  固定高さ
│  [FOLD]  [CHECK/CALL]  [RAISE]  │
│       [━━━━●━━━] スライダー      │
└─────────────────────────────────┘
```

**座席配置:**
- Seat 0 (自分/Hero): 画面下部中央
- Seat 1: 左側
- Seat 2: 対面（上部中央）
- Seat 3: 右側
- `viewingSeat`に応じて相対位置を自動計算

**PlayerSeatコンポーネント:**
- 名前、チップ残高、カード2枚、ベット額を表示
- アクティブプレイヤー: シアン枠ハイライト
- フォールド済: 50%透過でグレーアウト
- ディーラー: 「D」バッジ表示

**PlayingCardコンポーネント:**
- 表面: ランク + スート（♥♦=赤、♠♣=白/黒）
- 裏面: ダークグレー
- サイズ: 手札 25x35dp、コミュニティ 45x65dp

**CommunityCards:**
- 最大5枚を中央に横並び
- 未公開スロット: ダークグレー空枠

**PotDisplay:**
- ポット合計額（白太字）
- BB換算表示（グレー小文字）

## 7. アクションボタン

| 状況 | FOLD | 中央ボタン | 右ボタン |
|---|---|---|---|
| ベットなし | FOLD | CHECK | RAISE {額} |
| ベットあり | FOLD | CALL {額} | RAISE {額} |
| チップ不足でレイズ不可 | FOLD | CALL {額} | ALL IN {額} |
| 自分のターンでない | 全ボタン無効化（グレーアウト） |

**レイズスライダー:**
- 最小値: 現在のminRaise額
- 最大値: 自分の全チップ（= ALL IN）
- ステップ: BB単位
- 最大に合わせると「RAISE」→「ALL IN」表示切替

**バリデーション:**
- UI側でボタン活性/非活性を制御
- 最終バリデーションはGameEngine側（BettingRound）が担当
- 無効アクション時はUIで再入力を促す

## 10. デバッグモード / ホットシートモード

**ホットシートモード:**
- 常にアクティブプレイヤーの視点を表示
- 自分の手札のみ表、他プレイヤーは裏
- ターン切替時にインターステシャル画面を挟む:「端末を Player X に渡してください」
- タップで次のプレイヤーの視点に切替

**デバッグモード:**
- 全プレイヤーの手札が常に表（face-up）
- 視点固定（seat 0）、全席のアクションボタンが有効
- 画面上部にデバッグ情報を追加表示（フェーズ、ポット内訳、各ステータス）
- 一人でゲームの流れを全て確認可能

**モード切替:**
- LobbyScreenで選択、ゲーム中は変更不可
- GameContextの`mode`プロパティで管理
