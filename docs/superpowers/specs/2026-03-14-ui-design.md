# Jet Holdem - UI Implementation Design

## Overview

ゲームエンジン（Plan 1）完成済みの状態で、ローカルモード（BLEなし）で動作するUIを構築する。将来のBLE通信（Plan 2）追加時にUIを変更せずにGameServiceの実装を差し替えるだけで対応できる設計とする。

**スコープ:** Plan 3（UI）— Plan 2（BLE）・Plan 4（データ永続化）より先に実装する。

---

## 1. 技術スタック

| 項目 | 選定 | 理由 |
|---|---|---|
| ナビゲーション | Expo Router | Expoプロジェクト標準、ファイルベースルーティング |
| 状態管理 | React Context + useReducer | 外部ライブラリ不要、アプリ規模に十分 |
| テスト | React Native Testing Library + Jest | 既存Jest環境を活用 |
| スタイリング | React Native StyleSheet | 標準API、追加依存なし |

## 3. GameServiceインターフェース

UIとゲームロジックを疎結合にする抽象層。将来BleGameServiceに差し替え可能。

```typescript
// src/services/GameService.ts

interface ActionInfo {
  canCheck: boolean;
  callAmount: number;     // 0ならチェック可能
  minRaise: number;       // レイズ最小額（raise TOの値）
  maxRaise: number;       // = player.chips + player.bet（そのプレイヤー自身のbet）
  canRaise: boolean;      // チップがminRaise以上あるか
}

interface GameService {
  // 状態取得
  getState(): GameState;
  getActionInfo(seat: number): ActionInfo;  // UIのボタン表示用

  // アクション
  startGame(playerNames: string[], blinds: Blinds, initialChips: number): void;
  startRound(): void;
  handleAction(seat: number, action: PlayerAction): ActionResult;
  resolveShowdown(): ShowdownResult;
  prepareNextRound(): void;

  // 状態変更の購読（listenerはアクション後に同期的に呼ばれる）
  subscribe(listener: (state: GameState) => void): () => void;
}
```

**レイズ額のセマンティクス:** `handleAction`に渡すraise amountは「raise TO」（合計ベット額）。例: currentBetが100でraise to 300なら`{ action: 'raise', amount: 300 }`。スライダーの値もこの形式で渡す。

**LocalGameService:**
- `startGame()`はplayerNamesとinitialChipsからPlayer[]を構築し、新しいGameLoopインスタンスを生成する
- 全メソッドはGameLoopの対応メソッドをそのままラップ
- `getActionInfo()`はBettingRoundの内部状態から計算（minRaiseSizeのpublic getter追加が必要 → ゲームエンジンへの軽微な変更）
- subscribeのlistenerはアクション後に同期的に呼ばれる（Reactのstate更新バッチングに適合）

**将来のBleGameService:**
- BLE経由でGameState Notifyを受信 → subscriberに通知
- handleAction → BLE Write でホストに送信
- startGame/startRound等はホスト側のみ有効
- getActionInfo → GameStateから計算、またはホストから追加情報を受信

**ゲームエンジンへの必要な変更:**
- `BettingRound`に`get minRaise(): number`パブリックgetterを追加（現在privateのminRaiseSizeを公開）

## 4. GameContext

```typescript
// src/contexts/GameContext.tsx

interface GameContextValue {
  state: GameState | null;
  mode: 'hotseat' | 'debug';
  viewingSeat: number;           // 現在の視点プレイヤー
  service: GameService;
}
```

- コンポーネントは`service`のメソッドを直接呼び出す（dispatch/reducerパターンは不要）
- `mode`: ロビーで選択、ゲーム中は変更不可
- `viewingSeat`: ホットシートモードではアクティブプレイヤーに追従、デバッグモードでは固定（seat 0）

## 6. GameScreen レイアウト

flexboxベースでレスポンシブ対応。座標は比率ベース。

```
┌─────────────────────────────────┐
│  ステータスバー                    │
├─────────────────────────────────┤
│                                   │
│         Seat 2 (対面)             │  flex: 1
│                                   │  テーブル楕円エリア
│  Seat 1 (左)     Seat 3 (右)     │
│                                   │
│      ┌─────────────────┐        │
│      │ ポット / コミュニティ │        │
│      └─────────────────┘        │
│                                   │
│         Seat 0 (自分)             │
├─────────────────────────────────┤
│  アクションボタン                  │  固定高さ
│  [FOLD]  [CHECK/CALL]  [RAISE]  │
│       [━━━━●━━━] スライダー      │
└─────────────────────────────────┘
```

**座席配置（4人時）:**
- 画面下部 = viewingSeat（自分）
- 左側 = viewingSeatの次の席
- 対面 = viewingSeatの2つ先
- 右側 = viewingSeatの3つ先

viewingSeatが変わると全座席の画面上の位置がローテーションする（常に自分が画面下部に来る）。

**2人・3人時のレイアウト:**
- 2人: 下部（自分）と対面（相手）のみ。左右は空。
- 3人: 下部（自分）、左上、右上の3箇所。対面中央は空。

**PlayerSeatコンポーネント:**
- 名前、チップ残高、カード2枚、ベット額を表示
- アクティブプレイヤー: シアン枠ハイライト
- フォールド済: 50%透過でグレーアウト
- ディーラー: 「D」バッジ表示

**PlayingCardコンポーネント:**
- 表面: ランク + スート（♥♦=赤、♠♣=白/黒）
- 裏面: ダークグレー
- サイズ: 手札 25x35dp、コミュニティ 45x65dp

**CommunityCards:**
- 最大5枚を中央に横並び
- 未公開スロット: ダークグレー空枠

**PotDisplay:**
- `GameState.pots`（Pot[]配列）の合計額を算出して表示（白太字）
- BB換算表示（グレー小文字）
- サイドポットがある場合はResultOverlayで個別表示（ゲーム中はトータルのみ）

## 7. アクションボタン

| 状況 | FOLD | 中央ボタン | 右ボタン |
|---|---|---|---|
| ベットなし | FOLD | CHECK | RAISE {額} |
| ベットあり | FOLD | CALL {額} | RAISE {額} |
| チップ不足でレイズ不可 | FOLD | CALL {額} | ALL IN {額} |
| 自分のターンでない | 全ボタン無効化（グレーアウト） |

**レイズスライダー:**
- 最小値: `ActionInfo.minRaise`（raise TOの値）
- 最大値: `ActionInfo.maxRaise`（= ALL IN額）
- ステップ: BB単位
- 最大に合わせると「RAISE」→「ALL IN」表示切替
- スライダーの値はそのまま`{ action: 'raise', amount: sliderValue }`として送信

**バリデーション:**
- UI側は`getActionInfo()`を使ってボタンの活性/非活性を制御
- 最終バリデーションはGameEngine側（BettingRound）が担当
- 無効アクション時はボタン下部に赤文字でエラーメッセージを一時表示（3秒で自動消去）
- `ActionResult.reason`はエンジン内部向けの表現のため、`LocalGameService`でユーザー向けメッセージに変換して返す

## 8. ResultOverlay（ラウンド結果モーダル）

```
┌───────────────────────────────┐
│                                │
│     Winner: Player 1           │  白太字
│     Full House                 │  グレー（役名）
│     Kings over Sevens          │  グレー（詳細）
│                                │
│  ── 全プレイヤーの手札公開 ──   │
│                                │
│  P0: [K♠][J♦]  One Pair      │
│  P1: [A♥][K♠]  Full House ★  │  勝者ハイライト
│  P2: [8♠][7♣]  (folded)      │  グレー表示
│  P3: [Q♦][J♦]  Two Pair      │
│                                │
│  ── ポット分配 ──              │  サイドポット時のみ
│  Main Pot: 400 → P1           │
│                                │
│       [次のラウンドへ]          │  緑ボタン
│                                │
└───────────────────────────────┘
```

**表示ルール:**
- フォールド済プレイヤー: 手札非公開、"(folded)" 表示
- 勝者: シアン枠ハイライト
- GAME_OVER時: 「次のラウンドへ」→「ロビーに戻る」に変更
- 「次のラウンドへ」ボタン押下時: `service.prepareNextRound()` → `service.startRound()` を順に呼ぶ
- 背景: 半透明オーバーレイ

**全員オールイン時の挙動:**
- 全員がオールイン（actionable playerが0人）の場合、GameLoopは自動的にflop→turn→river→showdownまで一気に進む
- UIはsubscribeで最終状態（showdown）を受信し、コミュニティカード5枚が一度に表示される
- 将来的にアニメーション対応する場合は、中間状態のバッファリングを検討

## 10. デバッグモード / ホットシートモード

**ホットシートモード:**
- 常にアクティブプレイヤーの視点を表示（viewingSeat = activePlayer）
- 自分の手札のみ表、他プレイヤーは裏
- ターン切替時の流れ:
  1. アクション実行後、次のプレイヤーが異なる場合にインターステシャル画面を表示
  2. インターステシャル: 画面全体を覆い「端末を Player X に渡してください」と表示（手札が見えないように）
  3. 画面タップで次のプレイヤーの視点に切替、手札が表示される
- インターステシャルはコンポーネントとして`components/common/PassDeviceScreen.tsx`に実装
- showdown/roundEnd時はインターステシャルなし（全員に見せる情報のため）

**デバッグモード:**
- 全プレイヤーの手札が常に表（face-up）
- 視点固定（seat 0）、全席のアクションボタンが有効
- 画面上部にデバッグ情報を追加表示（フェーズ、ポット内訳、各ステータス）
- 一人でゲームの流れを全て確認可能

**モード切替:**
- LobbyScreenで選択、ゲーム中は変更不可
- GameContextの`mode`プロパティで管理
