Issues Found

1 上左コーナー(TL)が常に空 — 非対称レイアウト CRITICAL
SLOTS_BY_COUNT[9] には TL が含まれない。 9人でも topRow は [null, TC, TR] となり、上段が右寄りになる。 スマートフォンの小さな画面では視覚的バランスの崩れが目立つ。

修正案：9人テーブルでは TL を使い RB を外す (時計回り配置が自然に保たれる)。または TL に 装飾アイコン（ゲームロゴ等）を置いてデッドスペースを活用する。
2 compact席の幅52pxは実機で小さすぎる CRITICAL
compact時のPlayerSeatはpadding 4px(両端8px)、内部幅44pxのみ。

カード2枚だけで38pxを占め、名前テキスト(10px font)が4文字程度しか見えない。 現行の minWidth: 70px に対して 26%の縮小は実機での判読性に影響する。

推奨： compact を minWidth: 60px、padding 5px に見直す。
3 自分の席(BC)がopponentと同サイズ WARNING
compact時、compact={compact} はすべての席に等しく適用される。 しかしポーカーのUX原則として 自分の手札は常に目立つべき。 5人以上でもBC(viewer)は normal サイズのままにすることを推奨。

PlayerSeat に isViewer?: boolean prop を追加し、 viewer席のみ compact=false 固定にする案を検討する。
4 5〜8人でのEmptyスロットが視覚的に不整合 WARNING
6人ゲームの場合、topRowは [null, TC, TR]、 bottomRowは [BL, BC, null] となる。 つまり 上は右寄り、下は左寄り — 斜め対角に空きが生じる。 これはスマートフォン画面でアンバランスに見える。

6人レイアウトの見直し案：BL, BC, BR を底に揃え、 右サイドに RT, RB を使うなど対称性を優先する。
5 コミュニティカード幅がセンター幅とギリギリ WARNING
sideCol minWidth: 56px × 2 = 112px を引いた中央幅は 263px（375 − 112）。 コミュニティカード5枚は 5×45 + 4×4 = 241px を要する。 余白は わずか22px。

テーブルに paddingHorizontal: 8 があれば中央幅は247pxまで縮まり、 カードがはみ出す恐れがある。コミュニティカードを flex-shrink 対応にするか、 サイズを小さくする（例：40×58px）安全策が望ましい。
6 ランドスケープ(横向き)の考慮なし WARNING
6〜9人ゲームでは横向きの方がプレイしやすい場合が多い。 計画にはランドスケープレイアウトへの言及がない。

少なくとも 縦固定 (portrait lock) の明示、 または横向き時の代替レイアウトを検討する必要がある。
✓ 設計として正しい部分 GOOD
• スロットマッピング方式: getTableSlots() の純関数設計は テストしやすく拡張性も高い。
• 3行構造: top / middle / bottom の縦分割はポートレートスマホに自然にフィットする。
• 時計回り配置: 視聴者からの相対位置で席を決める方式は直感的。
• BC = 自分: 常に画面下中央に自分の席を置くのはポーカーUI標準に沿っている。

総評：修正なしでリリースは推奨しない

スマートフォンサイズでの主な問題は compact席の52px幅（Issue 2） と TLコーナーの非対称（Issue 1）。 特に52pxのコンパクト席は実機確認が必須。 コミュニティカードのフィット問題（Issue 5）も paddingの有無で本番クラッシュにつながりうる。

最優先修正 2点：
1. containerCompact.minWidth を 52 → 60 に変更し実機テスト
2. 9人の SLOTS_BY_COUNT を見直し TL を含める or 対称にする
